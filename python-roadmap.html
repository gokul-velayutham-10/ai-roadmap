🔄 Generators & Iterators
LLMs stream tokens one-by-one. Every streaming response in Python uses generators. You must understand yield deeply.
  • ▸ yield vs return — how generators pause MUST
  • ▸ yield from — delegating to sub-generators MUST
  • ▸ Generator expressions vs list comprehensions MUST
  • ▸ __iter__ and __next__ protocol GOOD
  • ▸ Lazy evaluation and memory efficiency GOOD
# LLM streaming pattern you'll use everywhere def stream_tokens(response): for chunk in response: yield chunk.delta.content # ← generator! for token in stream_tokens(llm_response): print(token, end="", flush=True)
🎨 Decorators & Closures
LangChain, FastAPI, and retry libraries are decorator-heavy. You'll constantly see @chain, @tool, @app.get — understanding them unlocks the whole ecosystem.
  • ▸ First-class functions — functions as arguments MUST
  • ▸ Writing your own decorators with functools.wraps MUST
  • ▸ Decorators with arguments (factory pattern) MUST
  • ▸ Closures and nonlocal scope GOOD
  • ▸ Class-based decorators GOOD
# LangChain uses this pattern extensively from langchain_core.tools import tool @tool def search_web(query: str) -> str: """Search the internet for current info.""" return tavily_search(query) # ← decorator wraps this
📦 Context Managers
Database connections, file handles, HTTP sessions in AI apps all use context managers. FastAPI's lifespan, database sessions, and model loading all use with statements.
  • ▸ __enter__ and __exit__ protocol MUST
  • ▸ @contextmanager decorator shortcut MUST
  • ▸ Async context managers (async with) MUST
  • ▸ Nested context managers and contextlib.ExitStack GOOD
🧬 OOP & Dataclasses
HuggingFace, LangChain, and Pydantic are class-heavy. You need to understand inheritance, dunder methods, and dataclasses to read and extend library code.
  • ▸ Dunder/magic methods: __repr__, __call__, __getitem__ MUST
  • ▸ @dataclass — auto-generated boilerplate MUST
  • ▸ Properties: @property getter/setter MUST
  • ▸ Class methods vs static methods vs instance methods MUST
  • ▸ Abstract base classes (ABC) GOOD
⚙️ Python Environment & Packaging

  • ▸ Docker for Python AI apps MUST

🔢 NumPy Essentials
Embeddings ARE NumPy arrays. Every vector operation — similarity, distance, normalization — uses NumPy. You can't avoid it.
  • ▸ ndarray creation: zeros, ones, random, arange MUST
  • ▸ Shape, reshape, transpose — critical for embedding dims MUST
  • ▸ Broadcasting rules — why shapes need to align MUST
  • ▸ Dot product, matrix multiply (@ operator) MUST
  • ▸ Vectorized ops vs Python loops (10–100x speed) MUST
  • ▸ Boolean indexing and fancy indexing MUST
  • ▸ Cosine similarity from scratch (you'll do this often) MUST
# Cosine similarity — what vector DBs do under the hood import numpy as np def cosine_sim(a, b): return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)) embedding_query = np.array([0.2, 0.8, 0.4]) embedding_chunk = np.array([0.3, 0.7, 0.5]) print(cosine_sim(embedding_query, embedding_chunk)) # 0.993

Async Python & HTTP CRITICAL
This is the #1 area JS developers underestimate in Python. Async in Python is different from JS Promises in important ways. LLM API calls, streaming, and concurrent batch processing all require confident async code.

🔀 asyncio Deep Dive
Every modern Python AI library is async-first. FastAPI, HTTPX, LangChain async methods, streaming — all asyncio. Coming from JS, the mental model is similar but the mechanics differ.
  • ▸ Event loop — how Python async differs from Node.js MUST
  • ▸ async def, await, coroutines MUST
  • ▸ asyncio.gather() — run multiple LLM calls in parallel MUST
  • ▸ asyncio.create_task() — fire and forget MUST
  • ▸ Async generators (async for) — streaming tokens MUST
  • ▸ Timeouts with asyncio.wait_for() MUST
  • ▸ Semaphore for rate-limiting parallel API calls MUST
# Batch 10 LLM calls in parallel — 10x faster than sequential import asyncio async def call_llm(prompt: str) -> str: response = await client.messages.create(...) return response.content[0].text async def batch_process(prompts: list[str]): sem = asyncio.Semaphore(5) # max 5 concurrent async def limited(p): async with sem: return await call_llm(p) return await asyncio.gather(*[limited(p) for p in prompts])
🌐 HTTPX & Streaming HTTP
LLM APIs use Server-Sent Events for streaming. You need HTTPX (async-capable) not requests for modern AI apps. Understanding SSE is critical for building streaming UIs.
  • ▸ HTTPX AsyncClient — async HTTP requests MUST
  • ▸ Streaming responses: iter_lines(), iter_bytes() MUST
  • ▸ Server-Sent Events (SSE) parsing MUST
  • ▸ Retry with exponential backoff (tenacity library) MUST
  • ▸ Connection pooling and session reuse GOOD
🚀 FastAPI for AI
FastAPI is the standard for AI microservices. You'll build LLM endpoints, webhook receivers, and streaming APIs. Its async-first design is perfect for AI workloads.
  • ▸ Route decorators: GET, POST, async handlers MUST
  • ▸ Request/Response models with Pydantic integration MUST
  • ▸ StreamingResponse — stream LLM tokens over HTTP MUST
  • ▸ Background tasks for async processing MUST
  • ▸ Dependency injection — clean API key handling MUST
  • ▸ Lifespan events — load models on startup GOOD
🔁 Error Handling & Retries
LLM APIs rate-limit, timeout, and return 500s regularly. Production AI apps must handle this gracefully. A 429 without a retry will crash your app.
  • ▸ Exception hierarchy — catching specific vs broad errors MUST
  • ▸ Tenacity library — @retry with exponential backoff MUST
  • ▸ Handling rate limits: 429 detection + wait logic MUST
  • ▸ Structured logging with structlog or loguru MUST
  • ▸ Circuit breaker pattern for cascading failures GOOD
⚡ PRACTICE PROJECT
Streaming FastAPI Endpoint
Build a FastAPI endpoint POST /chat that accepts a prompt, calls the OpenAI API asynchronously, and streams the response token-by-token as SSE. Add rate limiting with asyncio.Semaphore and retry on 429 with tenacity. This is the core of every AI product backend.

📐 Python Type Hints
LLM library code is heavily typed. You need to read and write types fluently to understand error messages and write library-quality code.
  • ▸ Built-in generics: list[str], dict[str, Any] MUST
  • ▸ Optional[X] = X | None — know both styles MUST
  • ▸ Union, Literal, TypedDict MUST
  • ▸ Callable, Generator, AsyncGenerator types MUST
  • ▸ TypeVar for generics — reading library code GOOD
  • ▸ Protocol — structural subtyping GOOD
⚙️ Pydantic v2
This is Python's Zod. Every LLM structured output, API schema, and config in LangChain, FastAPI, and the Instructor library is a Pydantic model. Master it.
  • ▸ BaseModel — defining schemas (= Zod z.object) MUST
  • ▸ Field validators: @field_validator, @model_validator MUST
  • ▸ Nested models — deep structured LLM output MUST
  • ▸ model_json_schema() — send schema to LLM for structured output MUST
  • ▸ Custom types with Annotated MUST
  • ▸ Settings management with pydantic-settings MUST
# LLM structured output — what Instructor library does from pydantic import BaseModel, Field class ExtractedInvoice(BaseModel): vendor: str amount: float = Field(gt=0, description="Total amount in USD") date: str line_items: list[str] # Claude returns JSON → Pydantic validates + parses it invoice = ExtractedInvoice.model_validate_json(llm_output)

Math Foundations MEDIUM — CONCEPTUAL ONLY
You don't need to derive backpropagation. But understanding vectors, probability, and a few key concepts will let you read AI papers, debug embedding issues, and make smarter architecture decisions.
↗ Linear Algebra (Practical)
Embeddings are vectors. Attention is matrix multiplication. You don't need to prove theorems — you need intuition for what these operations do.
  • ▸ Vectors and what they represent in high-dimensional space MUST
  • ▸ Dot product — why it measures similarity MUST
  • ▸ Matrix multiplication — conceptual understanding MUST
  • ▸ Norms — L1, L2 distance between embeddings MUST
  • ▸ Dimensionality — why 1536-dim embeddings exist MUST
  • ▸ Softmax — what it does in attention (intuition only) GOOD
🎲 Probability & Statistics
Temperature controls probability distributions. Perplexity measures model quality. Understanding basic probability makes you a smarter consumer of LLM outputs.
  • ▸ Probability distributions — what softmax output means MUST
  • ▸ Temperature & sampling — why higher = more random MUST
  • ▸ Mean, median, variance — for eval metric analysis MUST
  • ▸ Log probability — what LLMs actually output internally GOOD
  • ▸ Bayesian intuition — prior knowledge vs evidence GOOD
✓ Reality Check — Don't over-invest here
  • Spend 5–7 hours max. Watch 3Blue1Brown's "Essence of Linear Algebra" series (YouTube, free)
  • You need intuition, not derivations. If you can explain cosine similarity in plain English, you're good
  • Return here after Phase 2 — context makes the math click much faster

🧪 Testing AI Code
AI apps have two kinds of bugs: code bugs and model behavior bugs. You need pytest for code and a basic eval framework for LLM outputs. Companies hiring AI engineers test for this.

  • ▸ Writing LLM eval scripts — LLM-as-judge pattern MUST
  • ▸ Snapshot testing for prompt outputs GOOD
